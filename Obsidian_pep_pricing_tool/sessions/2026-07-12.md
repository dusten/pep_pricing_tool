# Session 2026-07-12

The heaviest data-integrity day of the run: a full-catalog duplicate-product audit and its merges, a three-way Lipo-C/MIC entanglement untangled from Claude call-log ground truth, TB-500 re-identified (twice — the first fix reversed itself), bulk CAS/MW research, the price-history tier bug, a vendor_sku uniqueness bug, and a run of Inventory/Files-tab features. Full detail: [[wiki/analyses/2026-07-12-duplicate-product-audit]], [[wiki/analyses/2026-07-12-product-cas-mw-research]], [[wiki/analyses/2026-07-12-tb500-b4-reidentification]], [[wiki/analyses/2026-07-12-price-history-tier-and-sku-collision]].

## Full-catalog duplicate-product audit → merges (#28, #57)

Read-only audit across all 205 products (normalized-name + substring heuristics, cross-referenced against existing aliases) found 3 HIGH-confidence pairs (merged same day, catalog 205→202) plus several MEDIUM candidates. A deeper follow-up pass using raw Claude extraction history (not just name comparison) upgraded 6 more to merge-recommended, approved and merged (202→196): Gonadorelin, Hexarelin, Adipotide/FTPP, Triptorelin Acetate, GLOW, Lipo-C with B12. Adipotide alone merged into Adipotide/FTPP the same day after user confirmation (196→195). Two items confirmed genuinely distinct molecules and left unmerged (NA Selank/Semax amidate vs. base forms, GHK-Cu vs. GHK-Cu Blend).

## Lipo-C/MIC three-way entanglement untangled

The MEDIUM-confidence Lipo-C/MIC merge exposed a deeper mess: reprocessing all 37 stored Claude call-log responses mentioning Lipo-C/MIC proved Claude's own `canonical_name` extraction was unreliable (the same vendor SKU extracted as two different product names across runs) — vendor SKU prefix turned out to be the reliable ground truth (LC120→no-B12, LC216→with-B12, LC396/425/526→the bigger FOCUS/FAT BLASTER blend). Fixed 5 misplaced rows + 9 exact-duplicate price rows across products 33/55, repointing cart/stack items first. Self-caught a bug in the migration itself (missed updating `pc_prices`' own denormalized `product_id` alongside the spec's) and folded the fix back into the archived script. Verified zero mismatches database-wide afterward.

## TB-500 re-identified as full-length B4 (reversing an earlier same-day fix)

Earlier the same day, a mislabeled alias was removed from TB-500 based on this wiki's own prior claim ("TB-500 usually means the fragment"). User pushed back with a stricter instruction: reprocess every vendor's raw extraction JSON and assume B4 (long form) unless a listing explicitly says fragment/17-23, or a second TB-500 line exists. Pulled all 35 call-log rows across 18 vendors — found the wiki's own prior claim was the actual root cause of the confusion. Re-identified product 2 as full-length B4 (correct CAS/MW), moved 3 fragment aliases + 2 misfiled price rows onto `TB500(Frag)` [359], and corrected the wiki page that had made the wrong blanket claim in the first place.

**Root cause, fixed**: the extraction prompt's own rule 8 asserted the same wrong fragment-default, nudging Claude toward guessed qualifiers on `canonical_name` instead of transcribing vendors' literal text — rewritten to drop the assumption and never fold qualifiers into the canonical name. Per-row ambiguity warnings (previously logged but never surfaced anywhere) now render as a ⚠ badge on pending Review Queue rows.

## Price-history tier bug + vendor_sku collision (backlog #59)

User spotted Purelypep Factory's KPV 10mg (1/10/50-kit tiers) showing what looked like one price oscillating — actually 3 independent tiers' changes interleaved, since `pc_price_history` never recorded which tier a change applied to (also silently corrupting Calendar's all-time-low milestones the same way). Migration 029 adds `tier_kit_size`, safely backfilling ~64% unambiguous rows; write path and Calendar queries fixed to match. A follow-up reconstruction pass recovered 1,087 more rows from raw Claude/pending-import data (92% total coverage) and surfaced a deeper bug along the way: `pc_prices`' uniqueness constraint didn't include `vendor_sku`, so a vendor listing the same product/spec/tier under two catalog codes had the second silently overwrite the first on every reimport (115 real collisions found). Fixed via migration 030.

## Inventory / Products / Files tab features

- **Hide price lines without deleting (#60)** — reused the pre-existing but never-individually-toggled `pc_prices.is_active`; fixed `commitPriceRow()` forcing it back to 1 on every reimport, which would have silently un-hidden a curated row.
- **$/unit on Inventory tab (#61)** — self-caught a real bug during verification: PDO returns DECIMAL columns as strings, and the new `.toFixed()` call crashed the entire admin panel render tree. Fixed by casting server-side.
- **DECIMAL-as-string audit (#63)** — swept the whole frontend (not just admin) for the same pattern after the crash above; found and fixed 2 more uncast instances in `ProductsTab.vue`'s backend.
- **Inline XLSX preview on Files tab (#62)** — same client-side pattern as the existing PDF preview; chose `exceljs` over the npm-registry `xlsx`/SheetJS package (2 unpatched HIGH-severity advisories with no fix available), since this parses vendor-submitted files.
- **Inline alias input (#64 start)** — replaced the native `prompt()` for "+ alias" with an inline text input, matching the tab's existing in-place-editing convention.

## Bulk CAS number / molecular weight research

Added `cas_number`/`molecular_weight` to `pc_products` (migration 028), CAS linking to PubChem on the Comparison page. Bulk-populated 117 of 195 products from web-verified sources (PubChem/DrugBank/ChemicalBook) across 4 parallel research passes — the rest deliberately left NULL (multi-ingredient blends, conflicting/unregistered CAS, heterogeneous biologics with no fixed MW). Caught and corrected a GHK-Cu/GHK-basic CAS mixup across research passes before writing to the DB.
