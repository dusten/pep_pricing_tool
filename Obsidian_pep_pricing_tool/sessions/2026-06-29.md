# Session Notes — 2026-06-29

## What Was Done

### First live deployment of price.themightygroupbuy.com

Phase 1 code was deployed to production for the first time. Multiple issues were found and fixed along the way.

---

## Fixes Applied (in order)

### 1. Missing admin tab stub files
`AdminView.vue` used `defineAsyncComponent(() => import('./tabs/SettingsTab.vue'))` but the files didn't exist. Vite resolves dynamic imports statically at build time — the `.catch()` fallback doesn't help. Created stub components:
- `frontend/src/views/admin/tabs/SettingsTab.vue`
- `frontend/src/views/admin/tabs/FeedbackTab.vue`

### 2. Deployment script overhaul (`add-price-site.sh`)
Multiple issues corrected:
- **Run as root**: removed all `sudo` prefixes; added root check at top
- **DB user renamed**: `pc_user` → `tmgb_price` to match server naming convention
- **Env file renamed**: `.env_price` → `.env_pricetool`, written to `~/.env_pricetool` (home dir) to match the server's existing `~/.env_peptools` pattern
- **Root password prompt**: AL2023 MariaDB root has a password set (set during peptools setup); passwordless socket auth didn't work
- **Schema import**: switched to import via root credentials rather than the new app user (avoids chicken-and-egg with GRANT)
- **File permissions**: `~/.env_pricetool` must be `640 ec2-user:apache` — PHP-FPM runs as `apache` and needs read access

### 3. Config and migration scripts updated
- `backend/config.php`: looks for `~/.env_pricetool` first (using `dirname(__DIR__, 2)`), falls back to app root for local dev
- `migrate.sh`: sources `~/.env_pricetool` with fallback to app root; default DB user updated to `tmgb_price`
- `deploy.sh`: rsync exclude updated from `.env_price` → `.env_pricetool`
- `.gitignore`: exclude updated to match

### 4. Apache vhost conflict
Certbot hijacked `peptools-le-ssl.conf` when adding the SSL cert for `price.themightygroupbuy.com` — it changed the `ServerName` to `price.themightygroupbuy.com` but kept the docroot pointing at the old peptools app. Result: HTTPS was serving the wrong app. Fixed by removing `peptools.conf` and `peptools-le-ssl.conf` from `/etc/httpd/conf.d/`. `price.conf` (which already had SSL hardcoded) became the sole vhost.

### 5. PHP spread operator syntax
`public/index.php` line 92 used `[$file, ...$paramNames]` (spread in array destructuring) which requires PHP 8.1+. The server's PHP-FPM was running an older version. Fixed by replacing with:
```php
foreach ($DYNAMIC as $pattern => $route) {
    [$file] = $route; $paramNames = array_slice($route, 1);
```

### 6. Vite base path
Built assets referenced `/assets/index-XXX.js` but the files live at `/dist/assets/index-XXX.js` (because `outDir` is `../public/dist`). Fix: added `base: '/dist/'` to `vite.config.js`.
